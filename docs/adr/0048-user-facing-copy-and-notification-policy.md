# ADR-0048: ユーザー向け文言と通知方針

## TL;DR

Discord の公開メッセージを主 UI とし、成功時の ephemeral 通知を減らす。メンションは「今すぐ行動が必要」または「開始直前」の通知に寄せ、文言は短く自然で非難しない日本語に統一する。

## Context

この Bot は固定 4 名の趣味仲間向けに、出欠確認・順延確認・開催決定・リマインドを Discord へ投稿する。既存実装は機能上は足りていたが、`OK` / `NG` / `見送り` / `スキップ` / `運用` などの語彙が混在し、金曜中止時に中止通知と順延確認の両方でメンションされ得る。

また、通常のボタン回答でも成功時に ephemeral follow-up を送っていた。回答状況は公開メッセージへ再描画されるため、成功通知を別途出すと Discord 上の小さな通知が増え、Bot がうるさく見えやすい。

UI 文言は Microsoft Writing Style Guide、GOV.UK content design、Material Design content guidance の共通原則に合わせ、短く、自然で、ユーザーが次の行動を判断できることを優先する。

## Decision

- ユーザー向け文言は feature-local な `messages.ts` と button constants に集約し、短く自然な日本語へ寄せる。
- 通常の時刻回答と順延 OK 回答では、公開メッセージの回答状況更新を主フィードバックとし、成功時 ephemeral follow-up を送らない。
- 欠席や順延不可のような不可逆に近い操作は、誤押下防止のため ephemeral 確認ダイアログを維持する。
- 権限外、期限切れ、古いボタン、内部エラーなど、他メンバーに共有する必要がない拒否/失敗は ephemeral を維持する。
- 金曜中止後に順延確認へ進む場合、中止通知側の全員メンションは外し、行動が必要な順延確認側で全員メンションする。
- 土曜中止など次アクションがない最終通知、開催決定、開始直前リマインドは、見落とし防止のため既存の対象メンションを維持する。
- 順延不可/不成立のカジュアルな表現は「お流れ」を維持し、`NG` はボタンや本文の主語彙から減らす。

## Consequences

- ボタン押下後の成功確認は、公開メッセージの回答状況更新で判断する。Discord の edit に失敗した場合は DB 正本のまま次の再描画で回復する既存方針を維持する。
- 金曜中止時のメンション回数が減り、直後の順延確認に注意を集めやすくなる。
- 文言変更は多数のテスト期待値に影響する。公開メッセージ、ボタンラベル、ephemeral 有無のテストを実装と同時に更新する。
- `requirements/base.md` の例示は、この方針に合わせて更新する。ただし締切、状態名、週キー、custom_id 形式などの SSoT は変更しない。

## Alternatives considered

- **すべての成功操作に ephemeral を出し続ける** — 押下直後の安心感はあるが、公開メッセージ更新と役割が重なり、通知が増えるため却下。
- **金曜中止通知と順延確認を 1 メッセージに統合する** — メッセージ数は減るが、既存の業務フローと復旧処理への影響が大きいため初手では却下。
- **「見送り」に統一する** — 丁寧だが趣味仲間向けにはやや硬く、既存の自然な「お流れ」を残したい意図と合わないため却下。

## Re-evaluation triggers

- ユーザーから、成功時に個別確認がないと不安だというフィードバックが継続して出たとき。
- 金曜中止と順延確認を別メッセージに分ける必要が薄れ、復旧処理も含めて安全に統合できる設計へ変えるとき。
- 対象メンバーやチャンネルが増え、固定 4 名前提のメンション方針では騒がしくなったとき。
